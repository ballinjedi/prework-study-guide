Prework Study Guide

HTML

  • HTML elements are a type of content on a webpage. Most commonly, they include: text, image, header and list elements.
  • Common elements used to divide and describe a page include: Meta-related elements, content division elements and semantic elements.
  • Each HTML element is composed of an opening and closing tag "<>"
  • At the top of an HTML document, you will see "!DOCTYPE html". That is a declaration that tells the browser what type of document it should expect when opened. In this case, HTML.
  • The head element contains information about the webpage.
  • The head element contains the following three elements: charset, which sets the character encoding; http-equiv, which sets an HTTP header for content; and viewport, which sets the default view for the webpage.
  • The title element defines the title for the webpage.
  • The style element defines the CSS styles associated with the webpage.
  • The base element defines the base URL for the webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, descriptions, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • The body element represents the visible content shown to the user.

CSS

  • A css rule contains a selector, which defines the element or attributes to which the rules or declarations will apply.
  • Declarations in a CSS rule contains two important components: the CSS property we want to apply and the value of the property.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • Git is a version control system that lets you manage and keep track of each version of your project. Git command cheatsheet
  • Repositories (or repos) are used as folders to store the following: files that contain code, files that contain other information about the code (e.g. a README file), and the version history for all files of the repo.
